## Monome Arc 4

4 endless rotary encoders with LED ring feedback. Connects via serialosc on port 17842.

### Encoder Mappings

**Primary layer:** Speed, Size, Density, Position

**Shift layer:** Jitter, Pitch, Filter Cutoff, Morph

**Looper mode:** When a looper voice is selected, encoders map to looper-specific controls.

LED rings display parameter values at 30 Hz.

---

## Monome Grid 128

16×8 button grid with variable-brightness LEDs. Connects via serialosc on port 17843.

### Layout

- **Step matrix (columns 0-7):** Step sequencer pattern. Each column = step, each row = note slot.
- **Control panel (columns 8-15):** Track selection, loop editing, chord mode, step/gate type selection.

LED output rendered at 30 Hz.

### Auto-Discovery

MIDI devices are detected automatically via CoreMIDI.

| CC | Parameter |
|---|---|
| CC 1 (Mod Wheel) | Macro Osc Morph |
| CC 74 (Brightness) | Macro Osc Timbre |

## AI Conversational Control

HTTP API for natural-language control via ChatGPT or Claude.

| Setting | Value |
|---|---|
| **Protocol** | HTTP/1.1 + WebSocket |
| **Address** | `127.0.0.1:4850` |
| **Base Path** | `/v1` |
| **Auth** | Bearer token |

Capabilities: session management, state reads, action bundles with timing/quantization, recording control, scene save/recall, undo/redo history, file search, WebSocket event stream.

For full API details, see [AI Control API](/docs/ai-api).

## Oscilloscope

The oscilloscope opens as a **floating window** and provides a real-time waveform display.

| Feature | Description |
|---|---|
| **Dual-source overlay** | Display two audio sources simultaneously for comparison |
| **Source selection** | Choose which engine/channel to visualize (includes Scramble gate, note, and mod sources) |
| **Time scale** | Adjust the horizontal zoom level |
| **Per-sample clock waveforms** | Visualize clock output signals alongside audio |

## Recording

### Master Output

| Setting | Value |
|---|---|
| **Format** | WAV (Linear PCM) |
| **Sample Rate** | 48 kHz |
| **Bit Depth** | 24-bit integer |
| **Channels** | Stereo |
| **Save Location** | `~/Music/Grainulator/Recordings/` |
| **Filename** | `{ProjectName}_{YYYY-MM-DD_HH-mm-ss}.wav` |

### Per-Voice Recording

Record into granular/looper buffers from external inputs, internal engines, or **master output** with one-shot or live loop mode. Slicer/looper voices default to **one-shot**, which normalizes the buffer length to the actual recorded content on stop.

## Project Management

Projects saved as `.grn` files with a sibling `audio/` folder. Legacy `.grainulator` format supported.

### Per-Project Audio Isolation

Each project maintains its own `audio/` directory alongside the `.grn` file. When you export buffers, record audio, or save reel data, files are written to this project-specific folder.

### What's Saved

A project snapshot includes all synthesis parameters, granular/looper voice parameters and audio buffer references, sequencer patterns (step, chord, drum — all 8 drum patterns + pattern sequencer steps), per-track and per-lane swing/humanize settings, Scramble state, mixer state, effects parameters, plugin host backend preference, insert/send plugin snapshots, clock output configuration, master clock settings, and scale/root note selection.

### Version History

The project format has evolved through 7 versions (v1-v7), with automatic migration from older formats. Version 7 adds: drum pattern bank (8 patterns), pattern sequencer, multi-level swing (global + per-track + per-drum-lane), swing algorithm/grid settings, and humanize parameters.
